# Issue #001 — 对话列表层级化：引入项目(Project)概念

**创建日期：** 2026-06-17
**状态：** ✅ Spec 已确认
**类型：** Feature

---

## 一、背景与动机

当前对话列表是一个扁平列表，用户所有对话混在一起。实际工作中，用户往往多项目并行（例如：A 市路网数据处理、B 市遥感影像分析、C 项目空间统计等），需在不同项目间频繁切换。

**目标：** 引入"项目（Project）"作为对话的容器，实现 Project → Conversation 的一层层级关系。支持项目归档，保持界面清爽。

---

## 二、数据模型

### 2.1 Project

```typescript
interface Project {
  id: string;        // 唯一标识（时间戳+随机串）
  title: string;     // 项目名称
  createdAt: number;
  updatedAt: number;
  archived: boolean; // 归档标记（无删除，只有归档）
}
```

### 2.2 Conversation（增量变更）

```typescript
interface Conversation {
  // === 已有字段 ===
  id: string;
  title: string;
  folderPath: string;
  createdAt: number;
  updatedAt: number;
  messages: unknown[];

  // === 新增 ===
  projectId: string; // 必填，一对一归属。每对话必属于一个项目
}
```

### 2.3 持久化结构

`conversations.json` 从 `Conversation[]` 升级为：

```json
{
  "projects": [
    { "id": "...", "title": "A市路网", "createdAt": ..., "updatedAt": ..., "archived": false }
  ],
  "conversations": [
    { "id": "...", "title": "路网转换", "folderPath": "/data/road", "projectId": "...", "createdAt": ..., "updatedAt": ..., "messages": [] }
  ]
}
```

> 兼容迁移：旧格式 `Conversation[]` 的 `conversations.json` 在加载时自动升级。

## 三、API 设计（IPC Handler）

### 3.1 新增 Handler

| Handler | 参数 | 返回 | 说明 |
|---|---|---|---|
| `get-projects` | 无 | `Project[]` | 获取全部项目 |
| `create-project` | `{ title: string }` | `Project` | 新建项目 |
| `rename-project` | `{ id: string, title: string }` | `void` | 重命名项目 |
| `archive-project` | `{ id: string }` | `void` | 归档项目（`archived=true`） |
| `unarchive-project` | `{ id: string }` | `void` | 取消归档（`archived=false`） |
| `move-conversation` | `{ convId: string, projectId?: string }` | `void` | 将对话移入/移出项目 |

### 3.2 修改现有 Handler

| Handler | 变更 |
|---|---|
| `create-conversation` | 新增可选参数 `projectId?: string` |
| `get-conversations` | 不变，仍返回全部 `Conversation[]`（不含 messages） |

### 3.3 Preload API（暴露给渲染进程）

```typescript
getProjects: () => Project[]
createProject: (title: string) => Project
renameProject: (id: string, title: string) => void
archiveProject: (id: string) => void
unarchiveProject: (id: string) => void
moveConversation: (convId: string, projectId?: string) => void
createConversation: (projectId?: string) => Conversation | null
```

## 四、UI 设计

### 4.1 布局

```
┌─────────────────────────────┐
│  对话列表          [+项目]   │  ← 全局仅有「新建项目」按钮
├─────────────────────────────┤
│  ▼ 🗂 A市路网             [+]│  ← 项目自带 [+]，直接建对话
│    │ 💬 路网格式转换         │  ← 对话条目缩进显示
│    │ 💬 路网拓扑检查         │
│  ▼ 🗂 B市遥感影像          [+]│
│    │ 💬 影像拼接             │
│                             │
│  ▶ 已归档 (2)               │  ← 折叠组
├─────────────────────────────┤
│  设置                       │
└─────────────────────────────┘
```

> 对话必须归属项目，无"未分类"概念。0 项目时，`[+]` 灰显，提示用户先创建项目。

### 4.2 交互行为

| 操作 | 触发方式 | 行为 |
|---|---|---|
| **新建项目** | 侧边栏头 [+项目] 按钮 | 弹出 prompt 输入项目名，创建后自动展开 |
| **新建对话** | 项目标题旁 [+] 按钮 | 打开目录选择框，创建对话并自动归属该项目 |
| **新建对话（0 项目时）** | 任意 [+] 不可用 | 提示"请先创建项目"，引导用户点击 [+项目] |
| **展开/折叠项目** | 点击项目标题 | 切换对话列表显示/隐藏，折叠状态持久化到 localStorage |
| **重命名项目** | 双击项目标题 | 显示内联 input，回车/blur 保存，Esc 取消 |
| **归档项目** | 项目标题右键菜单 → 归档 | 确认后设置 archived=true，项目移入已归档区 |
| **取消归档** | 已归档区展开，项目右键 → 恢复 | archived=false，项目回到活跃区 |
| **在已归档项目发消息** | 点击已归档项目下的对话，正常发送 | ***自动取消归档***，项目恢复到活跃区并展开 |
| **移动对话** | 对话右键菜单 → 移动到... → 选目标项目 | 更新 projectId，对话渲染到目标项目下 |
| **选择对话** | 点击对话条目（不变） | 切换到该对话 |
| **删除对话** | × 按钮（不变） | 确认删除 |
| **切换对话** | 点击（不变） | 加载对话消息 |

### 4.3 视觉规范

- **项目标题：** 字体略大于对话标题，粗体，左侧 `▼` / `▶` 折叠箭头
- **对话缩进：** 比项目标题缩进 16px，左侧竖线装饰
- **已归档项目：** 灰色文字
- **空状态：** 0 项目 + 0 对话时，"未分类"和其 [+] 依然显示，兜底保证用户能新建第一个对话
- **折叠状态：** 持久化到 localStorage

## 五、边界情况

| 场景 | 处理方式 |
|---|---|
| 旧版数据无 `projectId` | 迁移时自动创建"默认项目"，所有旧对话归入默认项目 |
| 旧版 `conversations.json` 是数组 | 检测格式，自动迁移为 `{ projects: [{ id, title: "默认项目", archived: false }], conversations: [..., projectId: "默认项目ID"] }` |
| 删除已归档项目下的对话 | 允许，行为不变 |
| 在已归档项目下发送消息 | 自动调用 `unarchive-project`，项目恢复到活跃区并展开 |
| 空状态（0 项目 + 0 对话） | 显示空列表提示"暂无项目，请点击 [+项目] 创建"，所有 [+] 灰显 |
